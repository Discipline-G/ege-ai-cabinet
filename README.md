# ПРОВЕРЯЙ

**ПРОВЕРЯЙ** — Telegram Web App и бот для учебной самопроверки сочинений ОГЭ/ЕГЭ по русскому языку.

Сервис помогает ученику быстро получить ориентировочную оценку по критериям, увидеть сильные и слабые места работы, получить подсветку проблемных фрагментов и краткую карту следующей работы.

> Важно: ПРОВЕРЯЙ не является образовательной организацией, официальным экспертом ФИПИ, школой, Рособрнадзором или экзаменационной комиссией. Проверка носит учебный и ориентировочный характер.

---

## Что умеет сервис

### Для ученика

- Проверка сочинений ОГЭ и ЕГЭ.
- Ввод работы текстом или фото.
- Ориентировочный балл по критериям.
- Разбор главных потерь.
- Подсветка наиболее уверенных проблемных фрагментов.
- Краткая карта следующей работы.
- Личный кабинет с балансом, историей и динамикой.
- Покупка пакетов проверок через T-Bank.

### Для преподавателя

- Учительский кабинет.
- Корпоративный баланс проверок.
- Ссылка для подключения учеников к классу.
- Просмотр работ учеников.
- Классная аналитика: количество работ, средний балл, динамика.
- Возможность выдавать проверки ученикам с общего корпоративного баланса.

## Экзамены и ограничения

### ОГЭ

- Проверяется сочинение ОГЭ.
- Релизное ограничение: до **350 слов**.
- Если тема или исходный текст не переданы, критерии содержания оцениваются ориентировочно по признакам внутри сочинения.
- Если работа меньше минимального объёма, сервис сообщает об ограничении и не должен оценивать её как полноценное сочинение.

### ЕГЭ beta

- Проверяется сочинение ЕГЭ в учебном beta-режиме.
- Релизное ограничение: до **450 слов**.
- Для более точной проверки желательно отправлять исходный текст задания и само сочинение.
- Если исходный текст не передан, К1–К3 оцениваются ориентировочно.

### Фото

- До **4 изображений** за одну отправку.
- Фото должно быть ровным, без бликов, сильного наклона и обрезанных строк.
- При рукописном тексте сервис проверяет работу по OCR-распознаванию, поэтому часть ошибок может быть связана с качеством фото.
- Если распознавание технически не удалось, проверка не списывается.

## Пользовательские сценарии

### Ученик

1. Открывает бота.
2. Выбирает ОГЭ или ЕГЭ.
3. Отправляет сочинение текстом или фото.
4. Получает:
   - результат;
   - критерии;
   - главные потери;
   - что исправить первым;
   - подсветку проблемных мест;
   - карту следующей работы.
5. При необходимости покупает проверки.
6. Смотрит историю и динамику в личном кабинете.

### Преподаватель

1. Открывает кабинет учителя.
2. Покупает или использует корпоративный пакет.
3. Копирует ссылку для учеников.
4. Отправляет ссылку в чат класса.
5. Ученики подключаются к классу.
6. Проверки учеников списываются с корпоративного баланса.
7. Работы появляются в кабинете учителя.

## Корпоративный счёт

Корпоративный счёт — общий баланс проверок для класса или группы учеников.

Как работает:

- преподаватель получает корпоративный код класса;
- ученики подключаются по ссылке вида `start=corp_...`;
- после подключения их проверки списываются с общего баланса;
- личные проверки ученика при этом сохраняются;
- работы подключённых учеников отображаются в кабинете преподавателя.

Если ученик уже подключён к другому преподавателю или классу, сервис не должен silently-перепривязывать его без поддержки или явного действия.

## Оплата

Оплата проходит через T-Bank acquiring.

Базовый поток:

1. Web App запрашивает создание платежа.
2. Backend создаёт `Init` в T-Bank.
3. T-Bank возвращает `PaymentURL`.
4. Пользователь оплачивает.
5. T-Bank отправляет webhook.
6. Backend проверяет подпись и статус.
7. При `CONFIRMED` начисляет проверки.

Проверки начисляются **после webhook от T-Bank**, а не в момент открытия страницы оплаты.

## Что не хранить в репозитории

Нельзя добавлять в GitHub:

- `.env`;
- токены Telegram;
- ключи OpenRouter/Yandex/T-Bank;
- `credentials_decrypted.json`;
- дампы PostgreSQL;
- реальные сочинения пользователей;
- скриншоты кабинетов с персональными данными;
- Telegram ID, username, webhook path или содержимое работ;
- приватные URL n8n, IP-адреса серверов и production-секреты.

## Переменные окружения

Переменные задаются на сервере/n8n. Значения не коммитятся.

Примерный список:

```env

TBANK_PASSWORD=
TBANK_API_BASE=
TBANK_NOTIFICATION_URL=
TBANK_SUCCESS_URL=
TBANK_FAIL_URL=
OPENROUTER_API_KEY=
YANDEX_OCR_API_KEY=
```

## Релизный smoke-test

Перед выкладкой новой версии:

### Оплата

- открывается страница T-Bank;
- после оплаты приходит webhook;
- проверки начисляются;
- баланс обновляется;
- повторный webhook не начисляет проверки второй раз.

### Проверка

- ОГЭ текстом проходит;
- ЕГЭ текстом проходит;
- фото проходит или даёт честный отказ без списания;
- отчёт содержит критерии, главные потери, подсветку и карту;
- проверка списывается только если отчёт выдан.

### Кабинеты

- личный кабинет показывает баланс и историю;
- учительский кабинет показывает работы учеников;
- корпоративная ссылка копируется;
- ученик подключается к классу по `corp_...`.

## Политика релиза

После запуска действует code freeze.

Без blocker-багов не трогать:

- оплату;
- T-Bank webhook;
- начисления проверок;
- списание проверок;
- основные n8n webhooks;
- scoring-критерии;
- корпоративную привязку.

Разрешены безопасные изменения:

- тексты;
- FAQ;
- подсказки;
- UI-кнопки;
- юридические страницы;
- README и документация.
