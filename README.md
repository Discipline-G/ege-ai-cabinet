# ПРОВЕРЯЙ — Telegram Web App кабинеты и юридические страницы

**ПРОВЕРЯЙ** — информационно-аналитический ИИ-сервис для учебной самопроверки сочинений ОГЭ/ЕГЭ по русскому языку.

Сервис принимает текст или фото сочинения, формирует учебный ориентировочный разбор, показывает вероятные ошибки, рекомендации, динамику и кабинет ученика/преподавателя.

> Важно: сервис не является образовательной организацией, официальным экспертом ФИПИ, школой, Рособрнадзором или экзаменационной комиссией. Результаты ИИ-проверки носят учебный и ориентировочный характер.

## Публичные страницы

- `index.html` — кабинет ученика.
- `teacher.html` — кабинет преподавателя.
- `pay.html` — страница оплаты / заявки на оплату.
- `faq.html` — FAQ и руководство пользователя.
- `offer.html` — публичная оферта.
- `user-agreement.html` — пользовательское соглашение.
- `privacy-policy.html` — политика обработки персональных данных.
- `personal-data-consent.html` — согласие на обработку персональных данных.
- `ai-disclaimer.html` — дисклеймер ИИ-проверки.
- `refund-rules.html` — правила оплаты, списаний и возвратов.
- `corporate-terms.html` — условия корпоративного тарифа.
- `pay-legal-block.html` — юридический блок для страницы оплаты.

## Что нельзя хранить в репозитории

Не добавлять в GitHub:

- токены Telegram, OpenRouter, Yandex, T-Банка;
- `.env`;
- `credentials_decrypted.json`;
- дампы PostgreSQL;
- бэкапы n8n;
- реальные сочинения пользователей;
- скриншоты кабинетов с персональными данными;
- логи с Telegram ID, username, токенами, webhook path или содержимым работ.

## Релизная проверка

Перед публикацией:

1. Проверить, что `faq.html` открывается.
2. Проверить, что `index.html`, `teacher.html`, `pay.html` ведут на актуальные документы.
3. Проверить, что в футере есть ссылки на оферту, соглашение, ПДн, согласие, дисклеймер, возвраты и поддержку.
4. Проверить, что в документах заполнены реквизиты ИП.
5. Проверить, что в маркетинговых текстах нет обещаний официальной оценки, гарантированного балла или связи с ФИПИ/Рособрнадзором.
6. Проверить, что в GitHub нет секретов и персональных данных.
7. После подключения эквайринга проверить оплату, чек, callback, начисление проверок и возврат.
